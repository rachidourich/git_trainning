Choosing a viewer

Some viewers are more appropriate than others for use in certain situations. Additionally, in some situations it may be best to use the ReportServerControl instead of the viewers. The viewers use the ReportServerControl internally to provide the capability to prompt for and pass information. If you prefer, you can use a ReportServerControl object for the same purpose without having to view the report. If viewing the report is not a concern, the simplest approach is to let the report viewer prompt for and pass information.
This table suggests the best viewer for several common usage scenarios.

Note: To display a report using any of the viewers, you must have the appropriate product activation keycode, as well as rights to view, refresh, edit, and export the report. For details, see the SAP BusinessObjects Business Intelligence Platform Administrator Guide.

Additional considerations when choosing a viewer
When choosing a viewer, you should also consider the report server (or servers) that you use.
The Crystal Reports 2011 Processing Server and the Crystal Reports 2011 Report Application Server provide full support for both the DHTML report page viewer and the report parts viewer.
