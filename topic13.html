Special viewer features

The viewers provide several special viewing capabilities including Locale-specific viewing, Embedded maps, Smart Tag parsing, and a tool panel that displays the group tree and the parameter panel.
Locale-specific viewing
To use this SDK to display reports in other languages when using a web browser, the international fonts must be installed on these machines:
  • The Crystal Reports 2011 Report Application Server or Crystal Reports 2011 Processing Server. xxxxx
  • The Java application server.
  • The machine running the web browser.
Although some report contents (such as the currency symbol and DateTime format) also rely on a locale setting, this setting does not need to be the same as the locale setting on the Crystal Reports 2011 Report Application Server or the Crystal Reports 2011 Processing Server.

Note: The currency symbol is determined by the locale setting on the server (not the user’s browser) to ensure that different users see the same currency figures.

Tip: It is possible for the default language setting of the web browser to be different from the system regional setting. For example, the system regional setting may be Chinese, but if the browser default language setting is English, the viewer is displayed in English. To set the system regional setting or the web browser’s language setting, consult the documentation for your operating system or web browser.
Embedded maps
The ability to view, and drill down on, embedded maps is offered by several of the viewers. This feature requires the mapping DLLs to be installed on the Crystal Reports 2011 Report Application Server or Crystal Reports 2011 Processing Server.
Smart Tag parsing
Crystal Reports lets users take advantage of the Smart Tags feature of Microsoft Office XP. With Smart Tags, users can copy a chart, text object, or field object from Crystal Reports and then paste the object as a Smart Tag into an Office XP application.

Note: The Smart Tag feature in Crystal Reports supports Word XP, Excel XP and Outlook XP. For Outlook XP support, Word XP must be set as the message editor.
The JSP page that is the target of a Smart Tag contains code that uses the CrystalReportSmartTagInfoParser Object. When a user clicks on the View the Report item in the Smart Tag Action menu, the Smart Tag passes information to the JSP page on the server. The CrystalReportSmartTagInfoParser Object then parses the information sent by the Smart Tag and the appropriate logon and report information can then be used to generate the report from within a web browser.
Tool panel
The tool panel appears on the left side of the viewer and can display the group tree or the parameter panel. You can hide the tool panel from the user programmatically.
Group tree
The group tree displays the headings for each group in the report in a hierarchy. Lists of nested group headings can be expanded or collapsed.
When the user clicks on a drilldown group, the drilldown view of the report on the selected group opens. When the user clicks on a group heading that does not represent a drilldown group, the viewer jumps to the page of the report where the selected group data is displayed.
The group tree view is specified with the CrToolPanelViewTypeEnum.groupTree enumeration.
Parameter panel
The parameter panel allows users to set static and dynamic parameters and to refresh the report. The parameter panel enables users to interactively format and filter report data. The panel displays all parameters that have been set to Editable or Read only. If a parameter is set to Editable, you can add, delete, or modify parameter values directly in the text box on the parameter panel.
Note: Advanced parameters include parameters that accept multiple or range values, parameters with a dynamic list of values, parameters that depend on other parameters, parameters with a large static list of values, and editmasks.
When you modify the value of a data parameter, the report refreshes and accesses the database to make the corresponding changes. When you modify the value of a non-data parameter, the report is formatted or filtered based on saved data within the report. The report does not access the database.
The parameter panel view is specified with the CrToolPanelViewTypeEnum.parameterPanel enumeration.
